# Paid weather sources analysis for Суперпрогноз от Serz

Дата: 28 апреля 2026

Важно: точные актуальные тарифы, лимиты и условия API перед подключением нужно отдельно проверить на сайтах провайдеров. Этот документ фиксирует архитектурный анализ и порядок подключения.

## Текущее состояние Serz

В приложении уже заложены бесплатные/открытые источники и модели:

1. Open-Meteo Best Match
2. GFS
3. ICON
4. ECMWF IFS через Open-Meteo, где доступно
5. UKMO через Open-Meteo, где доступно
6. MET Norway

Этого достаточно для хорошей бесплатной базы. Платные источники нужны не просто чтобы увеличить число источников, а чтобы добавить:

- более стабильный SLA;
- коммерческие условия использования;
- alerts / погодные предупреждения;
- nowcasting ближайших минут;
- исторические данные;
- более удобные текущие наблюдения;
- иногда более точные осадки по конкретной точке.

## Главный риск при подключении платных API

Платные API-ключи нельзя безопасно хранить прямо в Android APK. Их можно извлечь из приложения.

Правильная схема:

Android app → backend/proxy Serz → paid weather APIs

Backend должен:

- хранить API-ключи;
- кэшировать прогнозы;
- ограничивать частоту запросов;
- нормализовать ответы провайдеров;
- отдавать приложению единый формат;
- защищать бюджет от случайных/массовых запросов.

Без backend можно подключать только бесплатные публичные источники без секретных ключей.

## Кандидаты

### 1. Tomorrow.io

Сильные стороны:

- хороший кандидат для hyperlocal forecast;
- сильная сторона — осадки, nowcasting, предупреждения;
- удобно использовать как премиальный источник для ближайших часов;
- подходит для функции: “дождь через 30–60 минут”.

Слабые стороны:

- может быть дороже простых погодных API;
- нужно внимательно смотреть лимиты;
- без backend ключ нельзя класть в APK.

Роль в Serz:

- премиальный источник №1 для краткосрочного прогноза осадков;
- использовать не для всего прогноза, а прежде всего для nowcast/alerts.

Оценка приоритета: высокая.

### 2. OpenWeather / One Call

Сильные стороны:

- популярный и понятный API;
- удобные текущие данные, почасовой и дневной прогноз;
- есть погодные предупреждения в некоторых регионах;
- хорош как коммерческий базовый источник.

Слабые стороны:

- качество по точке зависит от региона;
- не всегда даст сильное преимущество над Open-Meteo по моделям;
- платные лимиты нужно контролировать.

Роль в Serz:

- коммерческий fallback;
- дополнительный голос в консенсусе;
- источник текущей погоды и alerts.

Оценка приоритета: высокая.

### 3. WeatherAPI.com

Сильные стороны:

- простой в подключении;
- часто удобен по структуре ответа;
- есть current/forecast/air quality/astronomy;
- хороший вариант как недорогой дополнительный источник.

Слабые стороны:

- не всегда самый сильный для сверхточного nowcasting;
- точность зависит от региона;
- нужен backend для ключа.

Роль в Serz:

- бюджетный платный источник;
- хорош для текущей погоды, воздуха, ежедневного прогноза и резервного сравнения.

Оценка приоритета: средне-высокая.

### 4. Visual Crossing

Сильные стороны:

- удобен для исторических данных;
- хорош для аналитики и сравнения прогноза с фактом;
- полезен для локальной памяти точности и ретроспективы.

Слабые стороны:

- не главный кандидат для “дождь через 40 минут”;
- больше аналитический источник, чем nowcast.

Роль в Serz:

- история погоды;
- проверка ошибок прогноза;
- обучение локальных весов источников.

Оценка приоритета: средняя.

### 5. Meteomatics

Сильные стороны:

- профессиональный погодный API;
- много параметров;
- высокая гибкость;
- подходит для серьёзной коммерческой версии.

Слабые стороны:

- обычно сложнее и дороже простых API;
- избыточен для первой пользовательской версии.

Роль в Serz:

- будущий professional / pro-provider;
- имеет смысл, если проект станет коммерческим.

Оценка приоритета: позже.

### 6. AccuWeather

Сильные стороны:

- известный бренд;
- есть прогнозы, текущая погода, alerts;
- узнаваемость для пользователей.

Слабые стороны:

- нужно внимательно изучать лицензии и ограничения;
- может быть не лучшим вариантом для старта из-за условий коммерческого использования.

Роль в Serz:

- брендовый коммерческий источник;
- можно рассматривать позже после проверки тарифов и лицензии.

Оценка приоритета: позже.

### 7. Foreca

Сильные стороны:

- сильный коммерческий погодный провайдер;
- может быть интересен для Европы;
- подходит для B2B-погодных решений.

Слабые стороны:

- условия и цена требуют отдельного запроса/проверки;
- может быть избыточен для первой версии.

Роль в Serz:

- B2B/профессиональный источник для будущей версии.

Оценка приоритета: позже.

## Рекомендуемый порядок подключения

### Этап 1: безопасная архитектура ключей

Перед платными API сделать backend/proxy.

Минимальный backend:

- endpoint: /api/weather/paid-consensus
- вход: latitude, longitude, city, mode
- выход: нормализованный JSON
- API-ключи: только в env backend, не в APK
- кэш: 10–30 минут
- лимиты: requests per device/day

### Этап 2: OpenWeather

Подключить как самый понятный коммерческий источник.

Что брать:

- current weather;
- hourly forecast;
- daily forecast;
- alerts, если доступны.

### Этап 3: Tomorrow.io

Подключить для главной функции “скоро дождь”.

Что брать:

- precipitation probability;
- precipitation intensity;
- nowcast / nearest hours;
- weather alerts.

### Этап 4: WeatherAPI.com

Подключить как резервный бюджетный источник.

Что брать:

- current;
- forecast;
- air quality;
- astronomy.

### Этап 5: Visual Crossing

Подключить для истории и проверки точности.

Что брать:

- historical weather;
- actual observations;
- сравнение с тем, что Serz обещал ранее.

## Лучшее сочетание для Serz

Оптимальный набор для первой платной версии:

1. Open-Meteo / GFS / ICON / ECMWF / UKMO / MET Norway — бесплатная база.
2. OpenWeather — коммерческий общий источник.
3. Tomorrow.io — премиум-осадки и ближайшие часы.
4. WeatherAPI.com — резерв и дополнительные параметры.
5. Visual Crossing — история и обучение точности.

## Не подключать всё сразу

Если подключить много платных источников без кэша и лимитов, приложение может быстро сжечь бюджет.

Правильная стратегия:

- Normal mode: бесплатные источники.
- Pro mode: бесплатные + OpenWeather.
- Max mode: бесплатные + OpenWeather + Tomorrow.io.
- Accuracy learning: Visual Crossing по расписанию/лимитам, а не на каждый экран.

## Итоговая рекомендация

Для ближайшего шага выбрать:

1. OpenWeather
2. Tomorrow.io
3. WeatherAPI.com

Но подключать их только через backend/proxy, не напрямую в Android APK.

Безопасный следующий технический шаг:

- добавить в проект документацию backend API;
- подготовить Kotlin-интерфейсы PaidWeatherProvider;
- не вставлять реальные ключи в приложение;
- сделать заглушки UI для платных источников;
- после появления ключей подключить backend.
